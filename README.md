* 자바에서 중요한 특징
        ** 다형성 (Polymorphism) : 하나의 타입이나 메서드가 여러 가지 형태로 동작하는 성질(하나가지고 여러방법으로 쓰겠다!) 
        ** 상속 (Inheritance) : 기존 클래스의 기능을 그대로 물려받아 새로운 클래스를 만드는 것
                ex)상위클래스-하위클래스 / * 클래스-메소드의 경우 포함관계이지 상속관계가 아님
                  ** 클래스(값을 나타내는영역) / 클래스(기능을 나타내는영역)
        ** 캡슐화 (Encapsulation) : 데이터(변수)와 그 데이터를 처리하는 함수(메서드)를 하나로 묶고, 외부에서 쉽게 접근하지 못하도록 숨기는 것
        ** 추상화 (abstraction) : 복잡한 실체에서 중요한것을 뽑아내어 단순화하는 것
  
* c언어 함수 vs 자바 메소드 : 절차지향(함수가 특정 클래스에 묶이지 않고 어디든 존재할수있음) vs 객체지향(반드시 특정 클래스안에 묶여야함)

* 자바 메소드 4가지 형태
       ** Type 1 (입력 X, 출력 X): "그냥 시키는 일만 해!"
              예: 단순히 고정된 문구 출력.
       ** Type 2 (입력 O, 출력 X): "재료 줄 테니 처리하고 끝내!"
              예: 이름을 전달하면 "OO님 환영합니다"라고 출력.
       ** Type 3 (입력 X, 출력 O): "재료는 필요 없으니 결과물만 가져와!"       
              예: 현재 시간을 알려주는 기능.
       ** Type 4 (입력 O, 출력 O): "재료 줄 테니 요리해서 다시 갖다줘!"
              예: 숫자 두 개를 주면 더한 값을 돌려주는 계산기.

* 자바의 자료형
       ** 클래스(Class) : 사용자가 정의한 객체 설계 (예: String, Scanner, 사용자 정의 클래스)
       ** 인터페이스(Interface) : 클래스가 구현해야 할 기능을 정의한 규격
       ** 배열(Array) : 같은 타입의 데이터 여러 개를 하나의 묶음으로 관리하는 구조
       ** 열거형(Enum) : 서로 연관된 상수들의 집합

* 변수, 문자상수, 문자열상수차이: 데이터를 담는 그릇인가, 고정된 값인가, 글자수에 따라 구분
       ** 변수 : a (이름으로 호출함, 값을 저장하는 메모리 공간(변경가능))
       ** 문자상수 : 'a' (딱 한글자의 고정된 )('ab' <- 이런건있나? 없다!)
       ** 문자열상수 : "a" (0개 이상의 글자묶)("ab" <- 존재함)

* 자바 식별자 규칙 : 대소문자구분, 첫글자제한(숫자x), 사용가능문자(영문자,숫자,밑줄,달러기호,한글도 사용가능(관습적으로 안씀)), 예약어 사용 불가, 공백 포함 불가

* 코드 생성 방법 예시
       ** 클래스 만들기 : public 선언하면 다른 클래스에서 접근가능, class 키워드로 클래스 선언, 클래스 코드는 {} 내에 모두 작성
       ** 주석문 : // (한 라인 주석) , /* */ (여러 행 주석)
       ** main() 메소드 : public static void으로 선언, String[] args로 실행인자를 전달받음
       ** 메소드 : C/C++에서의 함수를 메소드로 지칭, 클래스 바깥에 작성할 수 없음!(static 키워드 붙은 전역변수는 예외(진짜 바깥에 쓰는건아님,캡슐화의 원칙을 깨뜨리기 때문에 보통 쓸때는 final을 붙여서 씀))
       ** 메소드 호출 : sum() 메소드를 호출

* 식별자(identifier) : 클래스, 변수, 상수, 메소드 등에 붙이는 이름

* 좋은 이름 붙이는 언어 관습
       ** 가독성 높은이름(단축보다 풀네임(ex. s보다 sum))
       ** 충분히 긴 이름으로 붙이기(ex. AVM보다 AutoVendingMachine)
       ** 변수명은 소문자로 시작(기본자료형) & 대문자로 시작(참조자료형)       

* 자바의 데이터 타입
       ** 기본타입 : 8개(boolean, char, byte, short, int, long, float, double)
       ** 레퍼런스 타입 : 1개(용도는 3가지로 나눔)           
              *** 배열(array), 클래스(class), 인터페이스(interface)

* 명시적 형변환 / 묵시적 형변환 : 규칙은 큰 자료형을 따름 (작은데이터 -> 큰데이터(묵시적), (큰데이터-> 작은데이터(명시적, 무조건 타입을 써줘야함))

* 리터럴 : 프로그램에서 직접 표현한값, 정수&실수&문자&논리&문자열 리터럴있음
       ** 정수 리터럴 : 10진수,8진수(0),2진수(0b),16진수(0x)
       ** 실수 리터럴 : 소수점형태나 지수형태로 포함한 실수
              *** 실수 타입 리터럴은 double 타입으로 컴파일, 숫자뒤에 f(float), d(double)을 명시적으로 붙이기도 함
       ** 문자 리터럴 : 단일 인용부호('')로 문자 표현(ex. 'w' , 'A' , '가', '*', '3')
              *** 특수문자 리터럴은 백슬래시(\)로 시작
       ** 논리 리터럴 : true, false (boolean 타입 변수에 치환하거나 조건문에 이용)
       ** null 리터털 : 레퍼런스에 대입 사용
       ** 문자열 리터럴 : 스트링 리터럴로 이중 인용부호로 묶어 표현함 "" , 문자열 리터럴은 String 객레촐 자동 처리함
